Data Structures – Lecture 7: Trees & Graphs

Trees are hierarchical data structures with a root node and child nodes connected by edges. A Binary Search Tree (BST) ensures the left subtree contains nodes with smaller values and the right with larger, enabling O(log n) search. Graph traversal algorithms — BFS and DFS — differ in exploration strategy: BFS uses a queue (level-by-level), DFS uses a stack (depth-first). Time complexity for both is O(V + E) for adjacency lists.

🎯
Key Points
AI identified 5 points
  • BST property: left < root < right at every node level
  • BFS uses a Queue; explores nodes level by level (shortest path)
  • DFS uses a Stack (or recursion); explores full depth before backtracking
  • Graph complexity: O(V+E) with adjacency list, O(V²) with matrix
  • AVL trees maintain balance factor −1 to +1 via rotations

Original Notes 2,400 words · 8 min read

The Operating System acts as an intermediary between users and the computer hardware. The primary purpose of an OS is to provide an environment in which a user can execute programs conveniently and efficiently. An OS is software that manages the computer hardware. The hardware must provide appropriate mechanisms to ensure the correct operation of the computer system and to prevent user programs from interfering with the proper operation of the system. Process scheduling is an essential part of a Multiprogramming operating system. Such operating systems allow more than one process to be loaded into the executable memory at a time and the loaded process shares the CPU using time multiplexing. The scheduler is the component that facilitates this by tracking processes and selecting which to run using algorithms like FCFS, SJF, Round Robin, and Priority Scheduling…

Memory management is the functionality of an operating system which handles or manages primary memory and moves processes back and forth between main memory and disk during execution. Memory management keeps track of each and every memory location, regardless of either it is allocated to some process or it is free. It checks how much memory is to be allocated to processes and when this memory is to be freed or reallocated…

AI Summary ✦ 120 words · 30 sec read

An OS bridges users and hardware, managing processes, memory, and resources. Process scheduling (FCFS, SJF, Round Robin, Priority) enables multiprogramming by time-sharing the CPU. Memory management allocates and tracks RAM dynamically, moving processes between main memory and disk as needed.

▸OS = resource manager + environment for program execution
▸Scheduling algorithms determine CPU allocation order
▸Memory manager tracks free vs. allocated blocks at runtime
▸Paging & segmentation handle non-contiguous memory allocation
